What makes this magic?

If you are here, you probably already learned about shaders. If not, this is the technology rendering in real time the background behind this text. This projects makes it easy to launch even quite complex shaders as browser elements.

The GitHub repository documentation will guide you.

Features

  • simplicity: it is just rendering canvas background as fragment shader
  • speed: designed to be embedded in HTML and start rendering before other page resources are downloaded
  • extensibility: adding own interaction and controls is trivial
  • pixel feedback loops: preserving movement in time on offscreen buffers
  • shadertoy support: including multipass shaders
  • floating-point precision: for offscreen buffers and feedback
  • cross browser / cross device: on Chrome, Safari, Firefox or Edge, either with WebGL 1 or 2, on Linux, Windows, Mac, iPhone or Samsung phone. It will use optimal strategy to squeeze out what's possible from the browser and the hardware.
  • minimal footprint: transpiled from JavaScript to JavaScript with Closure Compiler
  • convenience: straightforward API, specific errors will inform you about mistakes which are otherwise hard to debug.

Why is it fast?

This website starts in milliseconds because there are no additional resources to be loaded — no extra code, no images. All the artwork and style is provided in the source code of this HTML page.

The shader will run on the GPU, not CPU, and contemporary devices, including our mobiles, already have one. Quite rich visual effects can be achieved with relatively efficient code.

Real time interactive shader offers a different perspective on compression — pure mathematics has orders of magnitude higher compression rate comparing to MPEG video. In contrary to video–loops, generatively synthesized video signal can be also interactive.

Alternatives

It is worth mentioning them here because. they brought the inspiration and they might be still more useful for your purposes:
